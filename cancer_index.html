Exploratory Data Analysis

What the data revealed

Analyzing 30 features to understand which ones are most predictive of cancer.

Top Cancer Predictors

  • worst concave points (0.79) — strongest signal
  • worst perimeter (0.78) — tumor size matters
  • mean concave points (0.77) — shape irregularity
  • worst radius (0.77) — larger = more likely cancer
  • mean perimeter (0.74) — consistent with radius

Key Insight

Size-related features (radius, perimeter, area) are the strongest predictors. Larger, more irregular tumors are more likely to be malignant. This aligns with medical knowledge — cancerous tumors tend to be bigger and have irregular shapes.

Multicollinearity

Many features are highly correlated with each other (radius, perimeter, area all measure size). Logistic Regression with StandardScaler handles this well as it normalizes all features to same scale.

No Missing Values

Dataset is clean — no imputation needed. All 569 patients have complete feature measurements. This is rare in medical datasets and allows us to focus on model building without extensive preprocessing.

ML Pipeline

From raw data to prediction

A clean sklearn Pipeline — preprocessing and model training in one object.

01

Data Loading & EDA

Loaded sklearn's built-in Breast Cancer dataset. Analyzed distributions, correlations, class balance.

569 rows, 30 cols
02

Train / Test Split

80/20 split with stratify=y to maintain class ratio. 455 train, 114 test.

455 / 114
03

Standard Scaling

All 30 features scaled to mean=0, std=1. Fit on train only, transform both train and test — no data leakage.

No leakage
04

Logistic Regression

Classification model. Linear equation → Sigmoid function → probability → threshold (0.3) → 0 or 1.

max_iter=1000
05

Threshold Tuning

Lowered threshold from 0.5 to 0.3 to eliminate False Negatives — no cancer patient should be missed.

FN = 0 ✅

Why Pipeline? Sklearn Pipeline chains preprocessing + model into one object. When we call pipeline.fit(X_train), it fits the scaler on train data only, then trains the model. When we call pipeline.predict(X_test), it applies the same scaling learned from train data to test data — preventing data leakage automatically.

Why threshold = 0.3? In cancer detection, a False Negative (missing actual cancer) is far more dangerous than a False Positive (unnecessary further tests). By lowering the threshold from 0.5 to 0.3, we eliminated all False Negatives — the model now catches 100% of cancer cases, at the cost of 1 extra false alarm. This is the right medical tradeoff.
